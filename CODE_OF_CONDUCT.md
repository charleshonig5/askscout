# Code of Conduct

## Standard

Contributors and maintainers are expected to act professionally. Be respectful, assume good intent, and engage with ideas rather than people.

This applies to issues, pull requests, discussions, commit messages, and any other project communication channel.

## Reporting

If someone's behavior is making the project a worse place to work in, email **charleshonigdesign@gmail.com** with the details. Reports stay confidential.

## Enforcement

The maintainer may, at their discretion: ask for behavior to change, edit or remove a comment, close an issue or pull request, or revoke contribution privileges. Severity-appropriate; first responses are usually private and informal.

## Scope

Project spaces (this repo, the askscout.dev support channels, any official communication) and any place a person is representing the project publicly.
